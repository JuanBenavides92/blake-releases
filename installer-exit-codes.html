Blake Restaurant Manager

Installer Exit Codes Documentation

Overview

This document describes the exit codes returned by the Blake Restaurant Manager installer (NSIS-based). These codes help system administrators and IT professionals diagnose installation issues.

Standard Exit Codes

Exit Code Status Description Action Required
0 Success Installation completed successfully None - Application installed correctly
1 Error Generic installation error Check installer log for details
2 Error User canceled installation Retry installation if needed
3 Error Fatal error during installation Check system requirements and permissions
4 Error Insufficient disk space Free up at least 500 MB of disk space
5 Error Access denied - insufficient permissions Run installer with appropriate user permissions
6 Error Another instance of installer is running Close other installer instances and retry

Updater Exit Codes

Blake uses a custom PowerShell-based updater. These are the exit codes from the update process:

Exit Code Status Description Action Required
0 Success Update completed successfully None - Application updated correctly
100 Error Update download failed Check internet connection and retry
101 Error Update extraction failed Check disk space and antivirus settings
102 Error Update installation failed Retry or download full installer
200 Warning Update skipped - user declined None - User chose not to update

Silent Installation

Blake supports silent installation with the following parameter:

Blake-Restaurant-Manager-Setup-1.1.59.exe /S
Note: In silent mode, the installer will use default settings and install to the standard location without user interaction.

Log Files

Installation logs are stored in:

  • Installer: %TEMP%\Blake-Restaurant-Manager-Install.log
  • Application: %APPDATA%\Blake Restaurant Manager\logs\
  • Updater: %TEMP%\BlakeUpdater.log

System Requirements

  • Windows 10 (64-bit) or later
  • 500 MB free disk space
  • Active internet connection
  • User-level permissions (admin not required)

Troubleshooting

Installation Fails with Exit Code 5

This usually indicates permission issues. Ensure:

  • You have write permissions to %LOCALAPPDATA%
  • Antivirus is not blocking the installer
  • Windows Defender SmartScreen is configured correctly

Update Fails with Exit Code 100

Download issues can be resolved by:

  • Checking internet connectivity
  • Verifying firewall/proxy settings
  • Manually downloading latest version from GitHub releases
